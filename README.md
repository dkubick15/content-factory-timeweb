# Content Factory - Timeweb Agent Edition

Личное приложение для подготовки статей и постов: база проекта, генерация,
редактор, обложки и публикация в Telegram сразу или по расписанию. Если
отдельный бот дублирует канал в Дзен, приложение остаётся единой точкой
публикации.

## Закрытый приватный режим (Timeweb AI Agent)

Сайт работает в закрытом режиме. Конечным пользователям больше не требуется вводить свои личные API-ключи или выбирать модели в интерфейсе. Все запросы на генерацию обрабатываются на стороне сервера через административную конфигурацию.

Если на сервере заданы переменные `TIMEWEB_API_KEY` и `TIMEWEB_AGENT_ID`, приложение автоматически переключается в полностью закрытый режим работы через ИИ-агента Timeweb Cloud.

## Основные особенности:
1. **Telegram-first**: формат материала может быть статьёй для Дзена или коротким постом, но канал публикации всегда Telegram.
2. **Чистый plain text**: генератор и публикация удаляют Markdown-маркеры, сохраняя обычные абзацы, списки и хэштеги.
3. **Медийный ИИ-критик**: оценивает попадание в аудиторию, удержание, пересылаемость, доказательства, CTA и потенциал охвата.
4. **Безопасность**: Ключ администратора никогда не передается в браузер пользователя. Он хранится исключительно на сервере в файле `.env`.
5. **Единый агент**: Все генерации контента проходят через обученного ИИ-агента, созданного в панели Timeweb Cloud.
6. **Серверное рабочее пространство**: проекты, идеи, медиа и очередь публикаций синхронизируются с сервером.
7. **Прод-защита**: включены безопасные HTTP-заголовки, лимиты на вход/генерацию/публикации и скрыт подробный health-вывод.
8. **Публикация по расписанию**: защищённый Worker забирает очередь каждую минуту и отправляет наступившие публикации в Telegram даже при закрытом браузере.
9. **ChatGPT App**: ChatGPT читает актуальный бриф Motor Port и по OAuth передаёт в редактор единый комплект «plain-text материал + сгенерированное изображение». Логин и пароль от ChatGPT приложению не нужны.
10. **Длинный текст с изображением**: материал до 4096 знаков публикуется одним Telegram-сообщением; для текста длиннее лимита подписи используется большой предпросмотр изображения над сообщением.

## Файлы в корне проекта:
- `index.html` - веб-интерфейс приложения
- `style.css` - все базовые стили интерфейса и адаптив
- `script.js` - клиентская логика, редактор и очередь
- `server.js` - серверная часть на Node.js (Express)
- `chatgpt-app.js` - OAuth 2.1 и MCP-инструменты личного ChatGPT App
- `oauth-login.html`, `oauth.css` - безопасный экран привязки ChatGPT к аккаунту приложения
- `package.json` - зависимости и скрипты запуска
- `.env.example` - шаблон настроек окружения
- `.gitignore` - игнорируемые git файлы
- `README.md` - общая документация
- `TIMEWEB_DEPLOY.md` - руководство по деплою в облако Timeweb

## Локальный запуск:
1. Установите зависимости:
   ```bash
   npm install
   ```
2. Скопируйте шаблон настроек окружения:
   ```bash
   cp .env.example .env
   ```
3. Заполните переменные в файле `.env`:
   - Для интеграции с Timeweb AI Agents добавьте ваш JWT-токен в `TIMEWEB_API_KEY` и ID вашего агента в `TIMEWEB_AGENT_ID`.
   - Для продакшена укажите `CORS_ORIGIN=https://ваш-домен.ru`.
   - Личный вход владельца включён по умолчанию: `kubik / kubik`. Его можно заменить через `DEMO_EMAIL` и `DEMO_PASSWORD` или отключить значением `ENABLE_DEMO_LOGIN=false`.
   - Если нужен подробный `/api/health` для диагностики, временно включите `DEBUG_HEALTH=true`.
4. Запустите сервер:
   ```bash
   npm start
   ```
5. Откройте приложение в браузере:
   ```text
   http://localhost:3000
   ```

## Подключение к ChatGPT

1. Откройте в приложении `Настройки` и скопируйте адрес ChatGPT App.
2. В ChatGPT включите Developer mode и создайте личное приложение по этому MCP URL.
3. При первом вызове войдите логином и паролем от «Контент-завода».
4. Попросите ChatGPT получить контекст Motor Port, подготовить текст и изображение, показать комплект и после одобрения передать его в «Контент-завод».

Пароль от ChatGPT нигде не используется. Записывающие MCP-инструменты защищены OAuth 2.1 с PKCE, а временная ссылка изображения скачивается только во время вызова.

## Референсы для развития

Эти репозитории использованы как рабочая референс-база по архитектуре и продуктовой логике:

- `gitroomhq/postiz-app` - сильный ориентир по social media scheduling: OAuth, платформы, команды, аналитика, API, очереди.
- `zernio-dev/latewiz` - близкий UX-референс: календарь, очередь, подключенные аккаунты, настройки площадок.
- `agenda/agenda` - референс для персистентных задач публикации вместо простого `setInterval`.
- `breejs/bree` - легкий вариант фоновых задач с worker threads, retry и graceful shutdown.
- `santiq/bulletproof-nodejs` - ориентир по разделению Node.js backend на routes/services/loaders.

## Следующий технический этап

Текущая версия остается компактным Node/Express-приложением. Чтобы превратить ее в устойчивую платформу, следующий крупный шаг лучше делать отдельной веткой:

1. Вынести backend из одного `server.js` в `src/routes`, `src/services`, `src/storage`.
2. Перейти с `data/users.json` на SQLite/PostgreSQL + Prisma.
3. Перенести данные приложения во внешнюю PostgreSQL, чтобы они не зависели от жизненного цикла контейнера.
4. Разделить фронт на компоненты или перейти на Next.js/React, если понадобится командная работа и сложный редактор.
